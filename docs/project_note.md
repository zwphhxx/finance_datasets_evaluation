# 面试展示说明

本文用于面试或代码走查时快速说明 财务/法律/投行场景大模型对比评测 的展示路径、核心边界和常见追问。它不是开发日志。

## 一句话定位

财务/法律/投行场景大模型对比评测是面向财务场景、法律场景和投行场景的模型评测样本库原型：通过样本维护、完整度校验、模型对比、专业标准答案、评分标准和人工复核，形成当前样本内的正式结论与使用边界。

## 推荐展示路径

1. 打开“项目说明”页，说明项目定位、评测闭环和当前展示边界。
2. 进入“样本库”，展示样本列表、选中样本详情和完整度检查。
3. 展开“样本库”，说明新增 / 编辑样本会在 SQLite 可用时同步正式评测资产。
4. 选择已入库且完整的样本，进入“发起评测”。
5. 选择多个模型，说明模型列表来自模型服务适配层，不硬编码模型名称。
6. 运行模型回答，说明被测模型只看到任务题、业务背景和输出要求。
7. 生成评分草稿，说明裁判模型才读取专业标准答案和评分标准。
8. 进入“评分确认”，查看评分矩阵、错误归因、红线提示和人工复核入口。
9. 人工确认或修订后，说明确认后才进入正式结论。
10. 进入“评测结论”，说明当前样本内的模型使用边界。

展示不要依赖固定样本、固定模型或固定分数。

## 关键讲解点

### 样本库不是普通题库

样本包含任务题、业务背景、专业标准答案、评分标准、历史信息和状态。完整且已入库的样本才能进入测试。

### 被测模型与裁判模型隔离

被测模型看不到专业标准答案、必须覆盖点、不可接受错误或评分标准。裁判评分链路才读取这些评判标准。裁判分数是建议分，不是最终结论。

### 人工复核是正式结论前置条件

评分草稿必须在“评分确认”页确认或修订后才进入正式结论。待确认草稿不会被结论页统计。

### 结论不是排行榜

模型使用边界只代表当前样本内观察。边界判断不只看平均分，还结合红线错误、关键维度短板、高风险任务表现、样本数量和人工复核说明。

边界表达统一为：

- 可作为初稿参考；
- 必须人工复核；
- 不可作为依据。

## 数据与运行边界

- 默认可从 `data/` 种子文件运行。
- SQLite 初始化后，正式评测资产优先从 SQLite 读取。
- `samples.json` 保留为轻量管理视图、导入导出和兼容备份。
- 未配置模型密钥时进入模拟回退模式。
- 模拟回退不代表真实模型结果。
- 不要提交 API Key，也不要在页面、日志或错误信息中展示认证请求头。

## 可回答的技术问题

| 问题 | 回答要点 |
| --- | --- |
| 为什么需要专业标准答案？ | 它是裁判评分和人工复核的评判锚点，被测模型不可见。 |
| 为什么需要评分标准？ | 它把专业判断拆成可复核的维度和扣分规则。 |
| 为什么需要人工复核？ | 高风险专业场景不能把裁判模型建议分直接当正式结论。 |
| 为什么不是排行榜？ | 样本规模和任务范围只支持当前样本内观察，不能代表模型整体能力。 |
| SQLite 与文件是什么关系？ | SQLite 可用时作为正式数据层；文件保留为种子数据、版本化资产和回退机制。 |
