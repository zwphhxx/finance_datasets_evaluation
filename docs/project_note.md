# 面试展示说明

本文用于面试或代码走查时快速说明“财务/法律/投行场景大模型对比评测”的展示路径、核心边界和常见追问。它不是开发日志。

## 一句话定位

这是一个面向财务场景、法律场景和投行场景的专业任务评测样本库原型：通过专业样本、专业标准答案、评分标准、裁判评分和人工确认，观察模型在当前样本内的回答质量、主要问题和使用边界。

## 推荐展示路径

1. 打开“项目说明”，说明项目定位、评测流程和数据边界。
2. 进入“样本库”，展示 13 条最终样本、筛选、样本详情和准入检查。
3. 在样本详情中说明任务题、业务背景、专业标准答案、评分标准和完整度如何支撑评测。
4. 进入“发起评测”，选择已入库且完整的样本。
5. 在“选择模型”弹窗中选择模型，并说明回答随机性对同一批评测统一生效。
6. 运行模型回答，说明被测模型只看到任务题、业务背景和输出要求。
7. 生成评分草稿，说明裁判模型才读取专业标准答案和评分标准。
8. 进入“评分确认”，查看当前评分摘要、评分依据和“查看评分材料”。
9. 人工确认、修订后确认或暂不采用评分草稿。
10. 进入“评测结论”，说明正式结论只汇总已确认评分。

展示不要依赖固定模型、固定分数或现场完整跑完全部模型调用。现场演示建议优先恢复已确认评分，再按需运行少量实时评测展示链路。

## 关键讲解点

### 样本库不是普通题库

样本不仅有题干，还包括业务背景、专业标准答案、必须覆盖点、不可接受错误、评分标准和状态。完整且已入库的样本才能进入发起评测。

### 被测模型与裁判模型隔离

被测模型看不到专业标准答案、必须覆盖点、不可接受错误或评分标准。裁判评分链路才读取这些评判材料。裁判分数是建议分，不是最终结论。

### 人工确认是正式结论前置条件

评分草稿必须在“评分确认”页人工确认或修订后确认，才进入正式结论。待确认、暂不采用和失败评分不会进入评测结论。

### 结论不是泛化排名

本项目不是模型排名页。模型使用边界只代表当前 13 条脱敏专业样本内观察，不能推广为对所有财务、法律或投行业务任务的稳定判断。

## 数据与运行边界

- 样本不包含真实公司、真实交易、真实个人或敏感数据。
- 本地数据文件是版本化样本资产；运行期数据库是页面执行、评分确认和结论恢复的承载层。
- 实时模型调用依赖外部服务，不能保证 100% 成功。
- 页面保留继续未完成项、重试失败项、评分恢复和已确认评分导入导出。
- 不提交 API Key、请求头、数据库文件、真实响应日志或临时导出。

## 可回答的技术问题

| 问题 | 回答要点 |
| --- | --- |
| 为什么需要专业标准答案？ | 它是裁判评分和人工复核的评判锚点，被测模型不可见。 |
| 为什么需要评分标准？ | 它把专业判断拆成可复核的维度、满分标准和扣分规则。 |
| 为什么需要人工确认？ | 高风险专业任务不能把裁判模型建议分直接当正式结论。 |
| 为什么不是模型排名？ | 样本规模和任务范围只支持当前样本内观察，不能代表模型整体能力。 |
| 现场演示如何稳定？ | 优先恢复已确认评分展示正式结论，再用少量样本实时跑通链路。 |
