# 面试演示脚本

## 30 秒开场

这个项目不是通用 Chatbot，也不是模型排名页。它是“财务/法律/投行专业任务大模型评测样本库原型”：用 13 条脱敏专业任务样本、专业标准答案、评分标准、裁判评分和人工确认，判断模型回答在当前样本范围内的质量、主要问题和使用边界。

## 推荐展示路径

1. **项目说明**：先讲项目定位、评测流程和数据边界。
2. **样本库**：展示最终 13 条样本，说明专业场景只有财务场景、法律场景、投行场景。
3. **样本详情**：打开一条样本，说明任务题、业务背景、专业标准答案、必须覆盖点、不可接受错误和评分标准如何构成评测底稿。
4. **发起评测**：选择少量样本和模型，展示被测模型只看到任务题、业务背景和输出要求。
5. **模型回答**：查看当前回答和技术明细入口，说明失败项可重试、已完成结果会保留。
6. **评分草稿**：生成裁判评分，查看复核提示和维度评分依据。
7. **评分确认**：人工确认、修订后确认或暂不采用，说明确认后才进入正式结论。
8. **评测结论**：展示已确认评分形成的当前样本内观察。

## 核心价值

- **专业样本**：任务来自财务核查、法律合规、投行判断等高风险场景。
- **标准答案**：每条样本都有标准结论、关键依据、必须覆盖点和不可接受错误。
- **评分标准**：评分维度有满分标准和扣分规则，避免只看主观印象。
- **输入隔离**：被测模型不看到专业标准答案、必须覆盖点、不可接受错误或评分标准。
- **人工确认**：裁判评分只是草稿，正式结论必须经过人工确认。
- **使用边界**：结论只代表当前已确认样本，不做脱离样本的泛化排名。

## 演示稳定性

现场演示建议优先使用已确认评分导入或恢复，先保证“评测结论”页能展示稳定结果。实时模型调用仍保留完整链路，但实时模型调用不能保证 100% 成功；网络、限流、模型响应时间和输出长度都可能导致单条失败。

如果现场要实时运行，建议选择 1-2 个样本和少量模型。页面默认支持批处理，不弹黄色警告；已完成回答会保留，未完成项可以继续，失败项可以单独重试。

## 常见追问

| 追问 | 回答 |
| --- | --- |
| 这是不是模型排名？ | 不是。它只说明当前样本内的回答质量、主要问题和使用边界。 |
| 被测模型会不会看到答案？ | 不会。被测模型只看到任务题、业务背景和输出要求。 |
| 裁判评分是否自动生效？ | 不会。裁判评分先进入评分草稿，人工确认后才进入正式结论。 |
| 为什么需要恢复已确认评分？ | 面试现场不应依赖外部 API 稳定性，恢复已确认评分能稳定展示最终闭环。 |
| 运行失败怎么处理？ | 单条失败不会影响已完成结果，可继续未完成项或重试失败项。 |
