# 样本质量门槛

本文规定样本进入正式测试前需要满足的最低质量要求。目标是保证每个样本都能支撑模型回答、裁判评分、人工复核和正式结论，而不是只维护一个题目列表。

## 测试准入门槛

| 门槛 | 要求 |
| --- | --- |
| 编号稳定 | 样本编号 / 任务编号存在且不可在编辑时修改。 |
| 任务题完整 | 存在任务题、业务背景和专业场景。 |
| 专业标准答案完整 | 存在标准结论、必须覆盖点和不可接受错误。 |
| 评分标准存在 | 评分维度和满分口径可被裁判评分链路读取。 |
| 状态正确 | 页面状态为已入库，底层状态为 `active`。 |
| 未移出测试 | 已移出测试或停用样本不可进入测试。 |

样本缺少关键字段时，可以保存为待复核或需优化，但不能进入“发起评测”页。

## 数据质量门槛

| 门槛 | 要求 |
| --- | --- |
| 脱敏合规 | 不写入真实公司、真实交易、个人信息或敏感数据。 |
| 可判定 | 任务应有明确判断标准，避免开放式主观题。 |
| 边界清楚 | 专业标准答案应说明适用边界和需要进一步核验的事项。 |
| 评分一致 | 评分标准扣分规则应足以支撑人工复核和裁判评分。 |
| 错误可追溯 | 错误标签应能指向具体错误表现和修正方向。 |
| 优化可验证 | 数据补强建议应说明补什么、如何观察是否改善。 |
| 正式结论审慎 | 结论只代表当前样本内观察，不做脱离样本范围的模型排名。 |

## 状态流转

| 页面状态 | 底层状态 | 含义 |
| --- | --- | --- |
| 待复核 | `draft` | 样本可编辑，不进入测试。 |
| 已入库 | `active` | 只有通过完整度校验后才进入测试。 |
| 需优化 | `draft` | 样本需要补充或修订，不进入测试。 |
| 已移出测试 | `inactive` | 样本停用，不进入测试。 |

## 入库前自检

1. 在“样本库”中查看样本详情，确认任务内容、专业标准答案、评分标准、历史信息和准入检查结构清楚。
2. 查看“样本完整度 / 入库检查”，确认缺失项已补齐。
3. 将状态设为“已入库”前，确认关键字段齐备。
4. 在“发起评测”页确认样本能被选择。
5. 生成评分草稿后，在“评分确认”页完成人工确认。

## 脚本校验

维护 `data/` 种子文件时，提交前运行：

```bash
python scripts/validate_dataset.py
```

脚本用于检查文件结构、专业标准答案完整度、评分标准权重、错误标签来源和对象关联。页面运行时的可测试判断以 `app/services/dataset_service.py` 的统一完整度校验为准。
