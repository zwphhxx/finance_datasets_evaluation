# 数据集质量门槛

本文件规定一道样本进入 FinDueEval 数据集前必须满足的质量门槛。门槛的目的不是堆数量，
而是保证每道样本都做到样本质量、边界控制、评分一致性、错误可追溯与数据补强可验证。
未全部满足的样本不予入库，先退回补全。

各门槛与 `scripts/validate_dataset.py` 的校验项一一对应，可在提交前自动核对。

## 门槛清单

| 门槛 | 要求 | 对应校验 |
| --- | --- | --- |
| 编号唯一 | `case_id` 在 `tasks.csv` 中唯一，沿用领域前缀命名 | case_id 唯一性检查 |
| 任务可判定 | 存在明确判定标准，能区分回答优劣，非开放主观题 | 人工评审 |
| 脱敏合规 | 不含真实公司、真实财务数据或个人信息 | 人工评审 |
| Gold Answer 完整 | 同时具备核心结论、关键依据、答案边界、不可接受错误 | Gold Answer 要素检查 |
| 可支撑评分 | 关键依据可逐条对照，足以支撑五维度 Rubric 判分 | Gold Answer 要素检查 |
| Rubric 完整 | 五个维度齐备，权重合计 100，扣分标准可执行 | Rubric 权重与字段检查 |
| 错误可追溯 | 错误标签均取自 `label_taxonomy.yml`，可归因到具体问题 | 错误标签来源检查 |
| 补强可验证 | 每条数据补强动作给出样本格式与验证指标 | 补强建议关联检查 |
| 关联完整 | 模型回答、评分可关联到有效 `case_id` 与已声明模型 | 外键关联检查 |
| 通过脚本校验 | `python scripts/validate_dataset.py` 输出无错误项 | 全部自动校验 |

## 关键门槛说明

### 答案边界

每道 Gold Answer 必须写明审慎边界与红线。边界用于约束“在什么条件下结论需谨慎、
何时应转人工复核”；红线用于标记不可接受错误（如未测算比例即下结论）。缺少边界的样本，
无法支撑“错误可归因”，视为不合格。

### 评分一致性

Rubric 每个维度都需写明扣分标准，使不同评审对同一回答给出接近的判分。维度分之和必须
等于 `total_score`，且每条评分附 `review_note` 扣分说明，便于复核与申诉。

### 错误可追溯与数据补强可验证

错误标签只能来自已登记的 taxonomy，保证同类问题在不同样本间口径一致；每条补强动作必须
带验证指标，使“补什么数据、补完如何确认生效”形成闭环，而不是停留在建议。

## 入库前自检顺序

1. 复制 `templates/new_case_template.yml` 填写并自检脱敏与判定标准。
2. 拆分写入 `data/` 下对应文件（流程见 `docs/dataset_building_guide.md`）。
3. 运行 `python scripts/validate_dataset.py`，确认无错误项、复核警告项。
4. 在「数据集质量与扩展框架」页面确认覆盖矩阵、Gold Answer 与 Rubric 质检结果符合预期。
