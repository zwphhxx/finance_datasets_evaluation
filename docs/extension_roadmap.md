# 扩展路线

财务/法律/投行场景大模型对比评测当前是面试展示原型。它已经具备样本库新增与导入、完整度校验、模型对比、裁判评分草稿、人工复核和正式结论链路；后续扩展应沿用现有数据结构和质量门槛，不通过新增复杂平台能力来掩盖样本质量问题。

## 当前阶段：面试展示原型

目标：

- 展示财务、法律、投行专业评测样本如何从任务题沉淀为完整评测资产。
- 证明被测模型与裁判模型输入隔离。
- 证明评分草稿必须人工复核后才进入正式结论。
- 用当前样本内观察形成审慎使用边界。

边界：

- 不包含真实公司数据。
- 不提供登录、权限、多人协作或外部数据库。
- 模拟回退只用于打通链路，不代表真实模型结果。
- 正式结论不是脱离样本范围的模型排名。

## 下一阶段：扩充样本覆盖

目标：

- 按模板或样本库新增与导入 增量加入更多脱敏任务。
- 补齐财务场景、法律场景、投行场景下的任务类型和风险等级。
- 每个新增样本都通过完整度校验。

重点不是堆数量，而是保证每个样本都有专业标准答案、评分标准、错误标签和可复核边界。

## 后续阶段：评审一致性

目标：

- 引入多名人工评审的独立复核记录。
- 观察同一模型回答在不同评审之间的分歧。
- 用分歧反向修订评分标准扣分规则和专业标准答案表述。

这一步可以在现有评分与复核结构上扩展，不需要改变当前主流程。

## 后续阶段：批量真实模型评测

目标：

- 基于现有 provider 逻辑批量运行更多模型。
- 保留运行批次、模型回答、裁判评分草稿和人工复核记录。
- 继续保持被测模型与裁判模型输入隔离。

真实模型结果仍需人工复核，不应因为运行规模扩大而跳过复核。

## 后续阶段：稳定评测集与报告

目标：

- 固化样本版本、评分标准口径和错误标签体系。
- 形成可复用的评测报告和数据优化建议。
- 用正式结论说明当前样本内的使用边界，而不是输出模型排名。

## 不在当前展示范围内

- 登录和权限系统；
- 外部协作数据库；
- 多人标注工作流；
- 企业级审计系统；
- 自动化采购建议；
- 将模拟回退结果当作真实模型结论。
