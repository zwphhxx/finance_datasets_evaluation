# 扩展路线

本文件说明 FinDueEval 从 MVP 扩展到正式评测集的路径。每个阶段都复用同一套数据结构、
质量门槛与校验脚本，因此当前虽是小样本 MVP，但扩展路径是确定的：增量加样本、加评审、
加真实模型，而不需要重构数据结构或页面逻辑。

阶段划分用于说明方向，不代表已完成的工作。当前仓库处于 MVP 阶段。

## MVP（当前）

- 范围：5 道脱敏专业任务，覆盖资本市场、财务尽调、法律审核、医学评估四个领域。
- 目标：验证「任务 → Gold Answer → 模型回答 → Rubric 评分 → 错误归因 → 数据补强 → 优化验证」闭环可运行。
- 交付：可本地运行、可部署的 Streamlit Demo；数据集清单、错误标签体系与校验脚本。
- 边界：样本量小，覆盖矩阵用于展示结构而非完整生产数据集。

## v0.2：扩充任务覆盖

- 范围：20–30 道任务，覆盖现有主要任务类型并补齐稀疏的领域 × 任务类型组合。
- 进入条件：每道新样本通过 `docs/dataset_quality_standard.md` 全部门槛与校验脚本。
- 交付：更密的覆盖矩阵；按领域/任务类型可观察的能力短板。
- 与当前差距：仅需按模板增量加样本，数据结构与页面不变。

## v0.3：引入多评审一致性

- 范围：同一回答由多名评审独立打分，记录评分一致性并对分歧项复核。
- 进入条件：Rubric 各维度扣分标准足够明确，可支撑跨评审复现。
- 交付：评分一致性指标；分歧样本的复核记录，用于校准扣分标准。
- 与当前差距：在现有 `scores.csv` 基础上增加评审维度，不改变评分口径定义。

## v0.4：增加真实模型批量评测

- 范围：以真实模型批量产出回答，替换当前脱敏模拟回答。
- 进入条件：模型先登记到数据集清单的模型范围；保留评测批次与提示词版本记录。
- 交付：真实回答的评分、错误标签与优化前后对比。
- 与当前差距：`model_outputs.csv` 接入来源由模拟改为真实采集，下游评分与归因流程不变。

## v1.0：稳定评测集与数据优化报告

- 范围：形成稳定的任务集、评分口径与错误标签体系，并产出可复用的数据优化报告。
- 进入条件：覆盖、评审一致性与真实模型评测均达到既定门槛。
- 交付：稳定评测集；从错误归因到数据补强再到优化验证的完整报告。
- 与当前差距：在前述阶段积累的样本与流程上固化口径，不引入新的数据结构。

## 为什么 MVP 已具备扩展基础

- 数据结构稳定：任务、Gold Answer、Rubric、模型回答、评分、错误标签、补强建议、优化对比
  各自独立成文件，新增样本不触碰页面逻辑。
- 质量门槛前置：每道样本入库前都要过同一套门槛，扩量不会稀释质量。
- 校验可自动化：`scripts/validate_dataset.py` 在每次新增后即可复核一致性。
- 标签与口径集中维护：错误标签体系与 Rubric 权重集中在清单与 taxonomy，扩展时口径一致。
